Scoring Rubric rubric.py
MetricWeightScoring MethodDetails
M1. Synthesis 5% Binary: 0 or 100 Did the generated code synthesize successfully?
M2. C-Simulation 10% Binary: 0 or 100 Did csim pass the testbench?
M3. Co-Simulation 10% Binary: 0 or 100 Did cosim (RTL simulation) pass?
M4. Latency (ns) 23% Ratio score: gen/GT ratio=1.0 → 85, better → 100, 2× worse → 42, 5× → 5. Uses latency_ns (accounts for Fmax).
M5. Fmax 10% Inverse ratio score Higher is better. Also checks timing slack (estimated period vs 4ns target).
M6. Resources 17% Weighted sub-metrics BRAM 30%, DSP 30%, FF 20%, LUT 20%. Each scored as gen/GT ratio.
M7. Area-Delay Product 15% ADP ratio score ADP = latency_ns × normalized_area. Combined efficiency metric.
M8. Device Feasibility 10% Utilization-based ≤50% → 100, 50-80% → 60-100, 80-100% → 20-60, >100% → 0 (infeasible)
Scoring Curve
Ratio → Score Function
Piecewise exponential decay:
ratio ≤ 1.0: score = 85 + 15 × (1 - ratio)
ratio > 1.0: score = 85 × e-0.7(ratio-1)

A ≥ 90 — Excellent
B ≥ 75 — Good
C ≥ 60 — Acceptable
D ≥ 40 — Below average
F < 40 — Poor
